# 初识 Vim

## 什么是 Vim

Vim 是最流行的文本编辑软件之一，有**编辑器之神**的美誉。社区活跃，插件丰富，生态优秀。许多 IDE 都支持 Vim 模式，如 VSCode、JetBrains 系列。

---

## Vim 的优势

适用于纯文本文件编辑（代码、Markdown、LaTeX 源码等），核心优势在于：

- 高效的移动
- 高效的编辑
- 高效的批量操作

---

## Vim 的四种模式

| 模式             | 说明                               |
| ---------------- | ---------------------------------- |
| **Normal 模式**  | 默认进入的模式，也是最常用的模式   |
| **Insert 模式**  | 插入模式，像普通文本编辑器一样输入 |
| **Command 模式** | 命令模式，在底部输入命令           |
| **Visual 模式**  | 可视模式，用于选择文本             |

---

## Normal 模式：基本移动

| 按键                | 功能                               |
| ------------------- | ---------------------------------- |
| `h` `j` `k` `l`     | 左 / 下 / 上 / 右                  |
| `gg`                | 跳到第一行（类似 Home 键）         |
| `G`                 | 跳到最后一行（类似 End 键）        |
| `{n}gg`             | 跳到第 n 行                        |
| `Ctrl-u` / `Ctrl-b` | 向上翻半页 / 一页（类似 PageUp）   |
| `Ctrl-d` / `Ctrl-f` | 向下翻半页 / 一页（类似 PageDown） |
| `zz` / `zt` / `zb`  | 光标行置于屏幕居中 / 顶部 / 底部   |

---

## 进入 Insert 模式

在 Normal 模式下，通过以下命令进入 Insert 模式：

| 按键 | 功能                                 |
| ---- | ------------------------------------ |
| `i`  | 在当前光标**之前**开始输入（insert） |
| `a`  | 在当前光标**之后**开始输入（append） |
| `o`  | 在**下方**插入新行，然后输入         |
| `s`  | **删除当前字符**，然后输入           |
| `I`  | 在**行首**开始输入                   |
| `A`  | 在**行尾**开始输入                   |
| `O`  | 在**上方**插入新行，然后输入         |
| `S`  | **删除当前行**，然后输入             |

> **记忆技巧**：大写与小写命令存在关联，可配对记忆（`i`/`I`，`a`/`A`，`o`/`O`，`s`/`S`）。

按 `Esc` 返回 Normal 模式。

---

## Command 模式

在 Normal 模式下按 `:` 进入 Command 模式：

| 命令           | 功能           |
| -------------- | -------------- |
| `:w`           | 保存当前文件   |
| `:q`           | 退出           |
| `:q!`          | 放弃更改并退出 |
| `:wq`          | 保存并退出     |
| `:h {command}` | 查看命令帮助   |

按 `Esc` 返回 Normal 模式。

---

## Visual 模式

在 Normal 模式下进入 Visual 模式：

| 按键 | 功能                                             |
| ---- | ------------------------------------------------ |
| `v`  | 进入**字符**可视模式                             |
| `V`  | 进入**行**可视模式（每次选中整行，适合多行选择） |

选中文本后的操作：

| 按键               | 功能         |
| ------------------ | ------------ |
| `x`                | 剪切选中内容 |
| `y`                | 复制选中内容 |
| `p`（Normal 模式） | 粘贴         |

按 `Esc` 返回 Normal 模式。

---

## Vim 与其他编辑器操作对比

| 操作        | 其他编辑器            | Vim                 |
| ----------- | --------------------- | ------------------- |
| 光标移动    | 方向键                | `h` `j` `k` `l`     |
| 文件首 / 尾 | `Home` / `End`        | `gg` / `G`          |
| 翻页        | `PageUp` / `PageDown` | `Ctrl-u` / `Ctrl-d` |
| 剪切        | 选中后 `Ctrl+X`       | 选中后 `x`          |
| 复制        | 选中后 `Ctrl+C`       | 选中后 `y`          |
| 粘贴        | `Ctrl+V`              | `p`                 |





# 移动与编辑

## 基于单词的移动

`hjkl` 仅能逐字符移动，效率有限。Vim 提供了基于单词的跳转命令：

| 按键 | 功能                                 |
| ---- | ------------------------------------ |
| `w`  | 跳到**下一个**单词的**开头**（word） |
| `b`  | 跳到**上一个**单词的**开头**（back） |
| `e`  | 跳到**下一个**单词的**结尾**（end）  |
| `ge` | 跳到**上一个**单词的**结尾**         |

> **大写版本** `W` `B` `E`：以连续非空字符为单词单位，跨度更大。

---

## 基于搜索的移动

### 行内搜索

| 按键      | 功能                             |
| --------- | -------------------------------- |
| `f{char}` | 跳到本行下一个 `char` **出现处** |
| `t{char}` | 跳到本行下一个 `char` **出现前** |
| `F{char}` | 向前版本的 `f`                   |
| `T{char}` | 向前版本的 `t`                   |
| `;`       | 重复上次 `f`/`t` 查找（向后）    |
| `,`       | 重复上次 `f`/`t` 查找（向前）    |

### 文件内搜索

| 按键         | 功能                                         |
| ------------ | -------------------------------------------- |
| `/{pattern}` | 跳到文件中**下一个** `pattern` 出现处        |
| `?{pattern}` | 跳到文件中**上一个** `pattern` 出现处        |
| `*`          | 搜索当前光标下的单词（等价于 `/{当前单词}`） |
| `n`          | 重复上次搜索（向后）                         |
| `N`          | 重复上次搜索（向前）                         |

> `pattern` 支持**正则表达式**。

---

## 基于标记的移动

| 按键          | 功能                               |
| ------------- | ---------------------------------- |
| `m{mark}`     | 将当前位置标记为 `mark`（`a`-`z`） |
| `` `{mark} `` | 跳转到名为 `mark` 的标记位置       |

**常用场景**：临时离开当前位置做一些操作后，快速跳回。习惯上可使用 `mm` 作为标记。

**内置标记**：

| 按键     | 功能                 |
| -------- | -------------------- |
| ` `` `   | 上次**跳转**前的位置 |
| `` `. `` | 上次**修改**的位置   |
| `` `^ `` | 上次**插入**的位置   |

---

## 其他实用跳转

| 按键 | 功能                             |
| ---- | -------------------------------- |
| `^`  | 跳到本行**开头**                 |
| `$`  | 跳到本行**结尾**                 |
| `%`  | 跳到匹配的**配对符**处（括号等） |

---

## Operator + Motion = Action

Vim 的核心编辑哲学：**`{operator}{motion}`** 组合出各种编辑动作。

### 常见操作符

| 操作符 | 含义                               |
| ------ | ---------------------------------- |
| `c`    | change：删除内容并进入 Insert 模式 |
| `d`    | delete：删除                       |
| `y`    | yank：复制                         |
| `v`    | visual：选中文本，进入 Visual 模式 |

### 组合示例

| 命令  | 效果                 |
| ----- | -------------------- |
| `dgg` | 删除到第一行         |
| `d$`  | 删除到行尾           |
| `dt;` | 删除直到分号前的内容 |
| `ye`  | 复制到单词结尾       |

> **操作符连按两次**（`dd` / `cc` / `yy`）：作用于**当前整行**。
> 例如 `dd` 删除当前行，`yy` 复制当前行。

---

## 重复操作

| 按键     | 功能           |
| -------- | -------------- |
| `.`      | 重复上一次修改 |
| `u`      | 撤销上一次修改 |
| `Ctrl-r` | 重做上一次修改 |

> `.` 命令非常适合需要**多次重复同一修改**的场景，省去重复输入，大幅提升效率。

---

## 批量操作：数字 + 动作

语法：**`{count}{action}`**，重复执行 `count` 次 `action`。

| 命令  | 效果          |
| ----- | ------------- |
| `4j`  | 向下移动 4 行 |
| `3dw` | 删除 3 个单词 |
| `2yy` | 复制 2 行     |
| `4p`  | 粘贴 4 次     |

**`.` 命令 vs 数字+动作**：
- `.` 命令：可以逐步观察每次变化，在合适时机停止；
- 数字+动作：需要预先确定次数，命令直观，语义明确。

---

## 技巧：相对行号

使用相对行号可以直观地确定行操作的范围，配合 `{count}{action}` 效果更佳。

| 命令                    | 功能         |
| ----------------------- | ------------ |
| `:set relativenumber`   | 开启相对行号 |
| `:set norelativenumber` | 关闭相对行号 |

---

## 小结

- 掌握进阶的移动命令（单词、搜索、标记）
- **（重要）** 理解 `{operator}{motion}` 的操作哲学
- 掌握用 `.` 命令重复上一次修改
- **（重要）** 掌握 `{count}{action}` 进行批量操作





# 文本对象操作

## 回顾

上一章介绍了 `[count]{operator}{motion}` 的操作逻辑——以当前光标与 `motion` 之间的区域作为操作对象。本章介绍另一种操作逻辑：**`{operator}{textobjects}`**。

---

## 文本对象

文本对象（textobjects）是具有**语义的文本片段**，格式为 `i`/`a` + 对象类型。

### 常见对象类型

| 对象                                  | 含义                 |
| ------------------------------------- | -------------------- |
| `w` / `W`                             | 单词                 |
| `s`                                   | 句子                 |
| `p`                                   | 段落                 |
| `(` `)` / `[` `]` / `{` `}` / `<` `>` | 对应配对符定义的区域 |
| `'` / `"`                             | 引号定义的区域       |

### `i` 与 `a` 的区别

| 前缀 | 含义          | 说明                       |
| ---- | ------------- | -------------------------- |
| `i`  | inner（内部） | 仅包含对象内部内容         |
| `a`  | a（一个）     | 额外包括周围的空格或配对符 |

---

## 文本对象操作示例

语法：**`[count]{operator}{textobjects}`**

| 命令  | 效果                 |
| ----- | -------------------- |
| `diw` | 删除一个单词         |
| `ci(` | 修改小括号内部的内容 |
| `yi{` | 复制大括号内部的内容 |

> 通过组合 `operator` 与 `textobjects`，可以对不同语义对象实施不同操作，灵活且易于记忆。配合 `.` 命令或 `[count]` 可轻松完成多次重复操作。

---

## textobjects vs motion

|              | `{operator}{motion}`               | `{operator}{textobjects}`    |
| ------------ | ---------------------------------- | ---------------------------- |
| 能否独立使用 | ✅ 可以（如 `w`、`b`、`e`）         | ❌ 不能，只能跟在 operator 后 |
| 范围确定方式 | 由光标移动确定，范围灵活但不够明确 | 显式指定操作对象，范围明确   |

两者都体现了 Vim **解耦操作与操作对象**的设计哲学，大大提升了编辑效率。

---

## 扩展插件

`operator`、`motion`、`textobjects` 均可通过插件扩展：

| 插件             | 功能                                      |
| ---------------- | ----------------------------------------- |
| `vim-easymotion` | 扩展 motion，提供更强大的跳转功能         |
| `vim-surround`   | 新增处理配对符（括号、引号等）的 operator |
| `vim-commentary` | 新增添加注释的 operator                   |
| `targets.vim`    | 扩展 textobjects，新增函数参数等文本对象  |

---

## 操作符与命令补充

| 命令         | 功能                         |
| ------------ | ---------------------------- |
| `gu{motion}` | 转换为小写                   |
| `gU{motion}` | 转换为大写                   |
| `g~{motion}` | 翻转大小写                   |
| `J`          | 将下一行合并到当前行（join） |
| `Ctrl-a`     | 光标处数字加一               |
| `Ctrl-x`     | 光标处数字减一               |
| `g Ctrl-a`   | 创建递增序列                 |
| `<`          | 左缩进                       |
| `>`          | 右缩进                       |

---

## 实用建议

### 让命令更模块化

尽量使命令具有**清晰的含义与明确的作用范围**，以便与 `.` 等命令协同使用。

> 例如：`daw` 比 `dw` 语义更清晰，也更模块化。

### 练习技巧：用 `v` 先可视化范围

初学时若不确定 motion 或 textobjects 的覆盖范围，可先用 `v{motion}` / `v{textobjects}` 将范围选中，再执行 `c` / `d` / `y` 等操作。

熟练后应尽量**直接使用完整命令**（如 `diw` 而非 `viwd`），这样更模块化，也更便于重复操作。

---

## 小结

- 了解常用的文本对象（单词、句子、段落、配对符等）
- **（重要）** 掌握 `{operator}{textobjects}` 的操作哲学
- 尽量使命令**模块化**，语义清晰，易于重复





# 寄存器与宏

## 寄存器

寄存器（Register）可以理解为 Vim 内置的**多个剪贴板**，一个字符对应一个寄存器。

### 常用寄存器

| 寄存器           | 含义                                     |
| ---------------- | ---------------------------------------- |
| `a`-`z`，`0`-`9` | 普通寄存器，供用户使用                   |
| `"`              | 默认寄存器，平时复制、删除的内容均存于此 |
| `%`              | 当前文件名                               |
| `.`              | 上一次插入的内容                         |
| `:`              | 上一次执行的命令                         |

通过 `:reg {register}` 查看对应寄存器中的内容。

---

## 指定寄存器

在复制、删除、粘贴等操作前加上 `"{register}`，即可指定本次操作使用的寄存器。

| 命令    | 效果                           |
| ------- | ------------------------------ |
| `"ayy`  | 将当前行复制到寄存器 `a`       |
| `"bdiw` | 删除当前单词，保存到寄存器 `b` |
| `"cp`   | 将寄存器 `c` 中的内容粘贴出来  |

**常见用途**：将需要持久保存的文本放入特定寄存器，随时粘贴，避免被默认寄存器覆盖。

> **大写寄存器**：使用大写字母（如 `"A`）表示**追加（append）**而非覆盖，将内容添加到对应寄存器的末尾。

---

## 宏

宏（Macro）可以**录制一系列键盘操作**，并在之后重放，操作序列存储在指定寄存器中。

### 基本用法

| 命令                 | 功能                              |
| -------------------- | --------------------------------- |
| `q{register}`        | 开始录制宏，存入寄存器 `register` |
| `q`                  | 停止录制                          |
| `@{register}`        | 重放寄存器 `register` 中的宏      |
| `@@`                 | 重放上一次执行的宏                |
| `[count]@{register}` | 重放宏 `count` 次                 |

**常见工作流**：`q{register}` 录制 → `@{register}` 重放 → `@@` 持续重放。

> **注意**：`.` 命令对宏**不生效**。`.` 只记录上一次单次修改，而宏可能包含多次修改，需用 `@@` 或 `[count]@{register}` 重放。

---

## 建议：让宏对连续重放友好

### 设计原则

在宏录制的**最后加入跳转到下一个编辑位置的动作**，使宏的结构为：

```
【编辑动作】 + 【跳转到下一个需要编辑的位置】
```

这样连续重放宏时，每次执行完编辑后光标自动移到下一处，从而对所有目标位置依次完成操作。

### 追加命令到已有宏

使用**大写寄存器**可在已录制的宏后面追加命令，而不必重新录制。

### 批量重放

若宏是重放友好的，可直接使用 `[count]@{register}` 一次性重放指定次数，高效完成批量操作。

---

## 小结

- 了解寄存器的概念及常用内置寄存器
- 掌握通过 `"{register}` 指定寄存器进行操作
- **（重要）** 掌握使用宏录制与重放，实现高效批量操作
- 设计宏时注意使其**对连续重放友好**



# 命令模式

## 概述

命令模式（Command-line mode）以**行**为基本操作单位，提供了 Normal 模式之外的另一套编辑方式。`normal` 和 `global` 命令可实现强大的批量行操作。

---

## Ex 命令格式

```
:[range] {excommand} [args]
```

| 部分        | 说明                           |
| ----------- | ------------------------------ |
| `range`     | 作用范围，不指定则默认为当前行 |
| `excommand` | Command 模式专用命令           |
| `args`      | 后续参数，视命令而定           |

### 常用 Ex 命令（`[x]` 为可选寄存器）

| 命令                  | 简写 | 功能                            |
| --------------------- | ---- | ------------------------------- |
| `:[range] delete [x]` | `d`  | 删除 range 中的行（到寄存器 x） |
| `:[range] yank [x]`   | `y`  | 复制 range 中的行（到寄存器 x） |
| `:[range] print`      | `p`  | 打印 range 中的行               |

---

## Address：指定位置

`address` 用于表示文件中的某一行：

| address       | 含义                                                  |
| ------------- | ----------------------------------------------------- |
| `{lineno}`    | 行号（如 `3` 表示第三行，`0` 表示第一行上方的虚拟行） |
| `$`           | 最后一行                                              |
| `.`           | 光标所在行                                            |
| `/{pattern}/` | 下一个 `pattern` 所在的行                             |

> address 支持**加减运算**：`.+3` 表示光标往下第三行，`$-3` 表示倒数第四行。

---

## Range：指定范围

`range` 由一个或两个 address 构成：`{address}` 或 `{address},{address}`

| range   | 含义                                                         |
| ------- | ------------------------------------------------------------ |
| `1,3`   | 第 1 到第 3 行                                               |
| `.,+4`  | 当前行到当前行往下第 4 行（共 5 行）                         |
| `$-3,$` | 最后 4 行                                                    |
| `%`     | 整个文件（特殊 range）                                       |
| `'<,'>` | Visual 模式选中区域的开头到结尾（Visual 模式下按 `:` 自动填入） |

### 示例

| 命令           | 效果                  |
| -------------- | --------------------- |
| `:1,3 delete`  | 删除第 1~3 行         |
| `:.,+4 yank`   | 复制当前行及往下 4 行 |
| `:$-3,$ print` | 打印最后 4 行         |

---

## 行的复制、移动与粘贴

| 命令                      | 功能                                     |
| ------------------------- | ---------------------------------------- |
| `:[range] copy {address}` | 将 range 中的行复制到 address **之后**   |
| `:[range] move {address}` | 将 range 中的行移动到 address **之后**   |
| `:[address] put [x]`      | 将寄存器 x 的内容粘贴到 address **之后** |

> 使用 `0` 作为虚拟行的 address，可将内容插入到**第一行之前**。

### Ex 命令 vs Normal 模式编辑

|                  | Normal 模式  | 命令模式   |
| ---------------- | ------------ | ---------- |
| 操作单位         | 字符/单词/行 | **行**     |
| 是否需要移动光标 | 需要         | **不需要** |

---

## 批量操作：`normal` 命令

```
:[range] normal {commands}
```

对 `range` 中的**每一行**执行 Normal 模式下的命令。

### 配合 `.` 命令

```
:[range] normal .
```

先手动执行一次修改，再用 `normal .` 在指定范围的所有行上重复该修改，**效果拔群**。

### 配合宏

```
:[range] normal @{register}
```

`.` 命令只能记录一次修改，若需要重放多步操作，可先将操作录制为宏，再用 `normal` 命令在指定行上批量重放。

---

## 批量操作：`global` 命令

```
:[range] global/{pattern}/[cmd]
```

对 `range` 中**包含 `pattern` 的所有行**执行 Ex 命令 `cmd`（不指定 `cmd` 则默认打印）。

### 与 `normal` 命令联用

```
:[range] global/{pattern}/normal {commands}
```

对范围内所有匹配行执行 Normal 模式命令，实现**按条件批量编辑**。

### 示例

| 命令                       | 效果                           |
| -------------------------- | ------------------------------ |
| `:% global/TODO/delete`    | 删除文件中所有包含 `TODO` 的行 |
| `:% global/TODO/normal @a` | 对所有含 `TODO` 的行重放宏 `a` |

## 替换命令

```
:[range]s/{pattern}/{string}/[flags]
```

将匹配 `pattern` 的内容替换为 `string`。

### flags

| flag | 含义                                               |
| ---- | -------------------------------------------------- |
| `g`  | 替换每一行中的**所有**匹配（默认只替换每行第一个） |
| `i`  | 忽略大小写                                         |
| `c`  | 替换前逐一**确认**                                 |
| `n`  | **计数**而不实际替换                               |

### 示例

```vim
:%s/Vim//gn
```

统计文件中 `Vim` 出现的总次数（加 `n` 后不会实际执行替换）。

## 一点人生的经验

> **Edit text at the speed of thought**

### 三条核心原则

**1. 分解操作**
将想要的编辑效果分解为**移动**与**编辑**的组合，而不是逐字符操作。

**2. 含义明确**
让每一条命令的含义清晰、作用范围明确，善用文本对象（`diw` 优于 `dw`），使命令模块化。

**3. 拥抱重复**
时刻思考：想要的效果能否分解为一系列**重复的操作**？Vim 对重复操作极其友好：

| 工具                        | 用途                 |
| --------------------------- | -------------------- |
| `.`                         | 重复上一次单次修改   |
| `[count]`                   | 直接重复指定次数     |
| `@@` / `[count]@{reg}`      | 重放宏               |
| `:[range] normal .`         | 对多行批量重复修改   |
| `:[range] global/{pat}/cmd` | 对匹配行批量执行命令 |
